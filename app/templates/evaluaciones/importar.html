Formato del JSON

El JSON debe contener solo las preguntas, con esta estructura:

{% raw %}{
  "preguntas": [
    {
      "enunciado": "¿Capital de Chile?",
      "tipo": "opcion_multiple",
      "alternativas": [
        { "texto": "Santiago", "es_correcta": true },
        { "texto": "Lima", "es_correcta": false }
      ]
    },
    {
      "enunciado": "El Sol es una estrella.",
      "tipo": "verdadero_falso",
      "alternativas": [
        { "texto": "Verdadero", "es_correcta": true },
        { "texto": "Falso", "es_correcta": false }
      ]
    }
  ]
}{% endraw %}

Reglas para que el JSON sea válido

  • El título y el umbral se escriben en el formulario, no en el JSON.
  • El JSON lleva una lista preguntas con al menos una.
  • Cada pregunta lleva enunciado y un tipo: "opcion_multiple" (de 2 a 6 alternativas) o "verdadero_falso" (exactamente 2).
  • En cada pregunta, una sola alternativa tiene "es_correcta": true.
  • En las verdadero_falso, la primera alternativa representa "Verdadero" y la segunda "Falso" (el texto lo fija la app). Marca como correcta la que corresponda: puede ser cualquiera de las dos.

¿No tienes las preguntas? Genéralas con una IA

{ "preguntas": [ { "enunciado": "...", "tipo": "opcion_multiple" | "verdadero_falso",
  "alternativas": [ { "texto": "...", "es_correcta": true|false } ] } ] }
